export function fib(n: i32): i64 {
  if (n < 2) return <i64>n
  return fib(n - 1) + fib(n - 2)
}
;; fib — recursion across the call/return boundary
(module
  (func $fib (export "fib") (param $n i32) (result i64)
    local.get $n
    i32.const 2
    i32.lt_s
    (if (result i64)
      (then
        local.get $n
        i64.extend_i32_s
      )
      (else
        local.get $n
        i32.const 1
        i32.sub
        call $fib
        local.get $n
        i32.const 2
        i32.sub
        call $fib
        i64.add
      )
    )
  )
)

A real engine, not a wrapper.

Everything from byte decoding to native code generation lives in Go. Nothing shells out, nothing links C.

Zero cgo

Every layer, from decoder to validator to codegen, is pure Go. No C toolchain, clean cross-compilation, one static binary.

Single-pass runtime

A single-pass x86-64 backend emits native code with microsecond compiles. A typed IR and an optimizing tier are on the roadmap.

Tracked conformance

Every applicable testsuite file runs on its own. The passing frontier is published in SPECTEST.md, not hidden.

Direct memory

mmap-backed linear memory is exposed straight to your code as a Go []byte, with no copies at the host boundary.

Stable host ABI

A stable wrapper ABI is generated for every export, keeping host ↔ wasm calls typed and low-latency across the boundary.

Open source

Apache-2.0 licensed and built in the open. Precompiled .wago blobs give fast reloads through the Go API.

Architecture

From bytes to native code

One front end decodes and type-checks every module. Today a single-pass backend emits x86-64 in one go; a planned typed IR adds an optimizing tier and more targets.

shared front end
→ →
↓ backend
Shipping

single-pass x86-64 

add eax, ecx   ; 01 c8

Straight to machine code in one pass — microsecond compiles, instant startup, native execution on an off-heap foreign stack.

Planned

typed IR → optimizing tier 

%2 = add.i32 %0, %1

A target-agnostic IR feeds a second tier — dead-code elimination, constant folding, register allocation — and new backends for arm64, macOS and Windows, still no cgo.
